# 联网补充来源策略

默认在所有非纯考试型问答中读取本文件，用于生成教材之后的官方网站补充层。
如果存在 `shared_memory/web_source_memory.json`，还要一起读取，用已经学到的域名偏好微调搜索顺序。

## 总原则

- 教材永远是主答案来源。
- 联网内容默认作为第二层补充延展，不替代教材结论。
- 优先使用 `官方原始来源`，其次才是 `高质量专业机构`。
- 不使用论坛、自媒体、问答社区、内容农场作为主依据。
- 如教材与联网资料冲突，必须明确提示：
  - `教材表述：...`
  - `联网补充：...`
  - `如用于考试，以教材为准`

## 站点优先级

### 第一优先级：官方标准与方法论拥有者

优先搜索这些站点：

- `pmi.org`
  - PMBOK、PMP、敏捷实践、项目组合/项目集相关官方资料
- `projectmanagement.com`
  - PMI 生态下的实践性补充内容；只作二级补充，不盖过 `pmi.org`
- `scrumguides.org`
  - Scrum Guide 官方文本
- `peoplecert.org`
  - PRINCE2、ITIL 等官方认证与方法论说明
- `axelos.com`
  - 旧版 PRINCE2 / ITIL 官方资料；若与 PeopleCert 重叠，优先 PeopleCert
- `iso.org`
  - ISO 标准官方说明
- `iec.ch`
  - IEC 标准官方说明

### 第二优先级：官方监管或政府站点

仅在问题涉及政策、法规、监管要求时使用：

- 中国政府与部委官网：如 `gov.cn`
- 国外官方监管机构官网：如 `sec.gov`、`europa.eu`

### 第三优先级：高质量专业组织

仅当前两级没有足够材料时使用：

- `agilealliance.org`
- 大学、研究机构、标准组织官网

## 禁用来源

以下来源不能作为主依据：

- 论坛、贴吧、知乎、博客园、CSDN、公众号、自媒体
- 无法确认作者和更新时间的转载站
- 只做 SEO 聚合、没有原始出处的内容站

## 默认触发联网补充的典型场景

- 所有概念解释题
- 所有章节总结题
- 所有比较题
- “按教材回答，再结合现实工作讲一下”
- “教材里这么写，那最新 PMI 官方怎么说”
- “这个概念在现实项目里一般怎么落地”
- “教材之外还有什么官方延展”
- “这个标准现在最新版强调什么”

## 应关闭联网补充的典型场景

- 用户明确说“只按教材回答”
- 用户明确说“不要联网”
- “按教材回答”
- 且同时带有“只按教材”“不要补充”的限定
- 纯选择题、填空题、简答题回忆

## 搜索与回答规则

### 共享来源记忆规则

- 如果 `shared_memory/web_source_memory.json` 中有 `observed_preferred_domains` 或 `observed_blocked_domains`，把它们当作最近学到的软提示。
- 如果其中还有达到阈值的高频 `preferred_domains` 或 `blocked_domains`，把它们当作稳定偏好优先执行。
- 如果其中有 `blocked_domains`，除非用户明确指定，否则不要用这些域名作为补充依据。
- 如果某个题型在 `preferred_by_query_type` 中已有稳定站点偏好，优先沿用。
- 用户一旦明确纠正“以后优先查哪个站 / 不要查哪个站”，应记录反馈并自动刷新共享来源记忆。

### 搜索规则

- 优先带域名过滤搜索，例如：
  - `site:pmi.org 项目章程 project charter`
  - `site:scrumguides.org scrum guide accountabilities`
- 如果题目问“最新”，优先看发布日期或版本号。
- 如果可以，只引用 1 到 2 个最相关的高质量来源，不要堆很多链接。

### 回答规则

- 先给 `教材答案`
- 再给 `联网补充`
- 再给 `来源`
- 最后在需要时加一句：`如用于考试，以教材表述为准`
- 如果确实没有找到足够高质量的官方来源，可以省略联网补充，但必须说明“未找到足够高质量的官方补充来源”

推荐输出格式：

```md
回答：
<教材结论>

补充依据：
- <教材依据>

引用页码：
- 第 X 页

联网补充：
- <官方补充点>

来源：
- PMI Official: <link>

说明：
- 如用于考试，以教材表述为准。
```
